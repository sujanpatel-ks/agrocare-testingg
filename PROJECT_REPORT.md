# 🌾 AgroCare AI - Project Documentation & Architecture Report

## 1. Executive Summary
**AgroCare AI** is a comprehensive, AI-powered smart farming assistant designed to empower farmers with real-time crop diagnosis, localized weather forecasts, market insights, and peer-to-peer community support. Built as a mobile-first web application, it bridges the technology gap for rural farmers by offering multi-language support (English, Hindi, Kannada) and accessible voice-based interactions.

## 2. Core Features & Modules

### A. AI Crop Disease Scanner
* **Functionality:** Users can upload or take a picture of a diseased crop leaf. The application analyzes the image and provides a detailed diagnosis.
* **Output Details:** 
  * Identifies the crop and the specific disease.
  * Provides a confidence score and bounding box of the affected area.
  * Lists symptoms and prevention methods (immediate and long-term).
  * Recommends treatments categorized into **Organic** and **Chemical** options, including dosage, frequency, precautions, and estimated costs.
* **Under the Hood:** Powered by the `gemini-3.1-pro-preview` multimodal AI model, which processes the base64 image data alongside a highly specific prompt to return structured JSON data.

### B. Voice-Enabled Smart Chatbot (AgroCare Bot)
* **Functionality:** A conversational AI assistant that helps farmers with general agricultural queries, treatment follow-ups, and farming best practices.
* **Voice Mode (Accessibility):**
  * **Speech-to-Text (STT):** Users can tap the microphone icon to speak their queries in their native language. Powered by the browser's native Web Speech API.
  * **Text-to-Speech (TTS):** Users can click "Listen" on any bot response to hear it spoken aloud. Powered by the `gemini-2.5-flash-preview-tts` model, which streams raw PCM audio data decoded and played via the Web Audio API (`AudioContext`) for robust playback.
* **Under the Hood:** The chat first attempts to route queries through a custom **n8n webhook** (`agrocare.app.n8n.cloud`). If the webhook is unavailable, it seamlessly falls back to the `gemini-3-flash-preview` model with a custom system instruction tailored for agricultural advice.

### C. Community Forum
* **Functionality:** A peer-to-peer discussion board where farmers can share experiences, ask questions, and discuss topics like pest control, fertilizers, and market trends.
* **Features:** 
  * Real-time search functionality that filters discussions by title, content, author, and tags.
  * Categorized tags (e.g., #Disease, #Fertilizer).
  * Quick access banner for Government Schemes and Subsidies.

### D. Weather & Location Services
* **Functionality:** Provides highly localized, real-time weather updates and a 5-day agricultural forecast.
* **Features:**
  * **Real-time Weather:** Temperature, humidity, rainfall, and wind speed.
  * **5-Day Forecast:** Includes specific agricultural advice based on upcoming weather conditions (e.g., "Delay pesticide application due to upcoming rain").
  * **Nearby Suppliers:** Locates nearby seed stores, fertilizer shops, and agricultural input suppliers.
* **Under the Hood:** Utilizes the browser's Geolocation API. The coordinates are fed into the `gemini-3-flash-preview` model utilizing **Google Search Grounding** (for weather) and **Google Maps Grounding** (for suppliers) to fetch real-time, accurate data.

## 3. Technical Architecture

### Frontend Stack
* **Framework:** React 18+ with Vite (Single Page Application).
* **Styling:** Tailwind CSS for responsive, mobile-first utility styling.
* **Animations:** `framer-motion` for smooth page transitions and micro-interactions.
* **Icons:** `lucide-react` for clean, consistent iconography.

### AI & Backend Integrations
* **Google Gen AI SDK (`@google/genai`):** The core engine driving the intelligence of the app.
  * *Vision:* `gemini-3.1-pro-preview` for crop image analysis.
  * *Text/Grounding:* `gemini-3-flash-preview` for chat fallback, weather, and supplier location.
  * *Audio:* `gemini-2.5-flash-preview-tts` for generating high-quality voice responses.
* **n8n Automation:** Acts as the primary backend logic layer for the chatbot, allowing for complex, visual workflow automation before falling back to direct LLM calls.

### Browser APIs Utilized
* **Geolocation API:** To fetch the user's latitude and longitude for weather and supplier mapping.
* **Web Speech API (`SpeechRecognition`):** For converting user voice input into text.
* **Web Audio API (`AudioContext`):** For decoding and playing raw binary audio buffers generated by the TTS model.

## 4. Data Flow Example: Crop Diagnosis
1. **Capture:** User takes a photo via the UI.
2. **Process:** The image is converted to a Base64 string.
3. **Request:** The app sends the Base64 string + a strict JSON-schema prompt to the Gemini Vision model.
4. **Response:** Gemini returns a structured JSON object containing the disease name, treatments, and translations.
5. **Render:** The React frontend parses the JSON and displays it in a user-friendly, tabbed interface (Organic vs. Chemical).

## 5. User Experience (UX) & Accessibility Focus
* **Multi-Lingual:** A global toggle allows users to instantly switch the interface and AI responses between English (EN), Hindi (HI), and Kannada (KN).
* **Color Psychology:** Uses earthy tones (`bg-soil`, `text-earth`) and agricultural greens (`bg-primary`) to create a familiar, trustworthy environment for farmers.
* **Illiteracy Mitigation:** The heavy integration of Voice Input (Mic) and Voice Output (Listen buttons) ensures that farmers who cannot read or write can still fully utilize the application.
